# Data Requirements Brief for Kigombya Primary School Database Project

**Project:** School Management Information System (SMIS)  
**Institution:** Kigombya Primary School, Mukono  
**SDG Alignment:** SDG 4 - Quality Education (Ensure inclusive and equitable quality education and promote lifelong learning opportunities for all)

---

## Purpose
This document outlines the data requirements needed to develop a comprehensive School Management Information System that will help improve administrative efficiency, student tracking, and educational outcomes at Kigombya Primary School.

---

## Data Categories Required

### 1. **Student Information**
- Student Registration Number (unique identifier)
- Full Name (First Name, Middle Name, Last Name)
- Date of Birth
- Gender
- Nationality
- Place of Birth
- Home Address (Village, Parish, Sub-county, District)
- Contact Information (Phone number, Email if available)
- Parent/Guardian Information (Name, Relationship, Contact)
- Enrollment Date
- Current Class/Grade
- Previous School (if transferred)
- Special Needs/Disabilities (if any)
- Medical Conditions/Allergies (if any)
- Photo (optional)

### 2. **Academic Records**
- Class/Grade Level (P.1, P.2, P.3, P.4, P.5, P.6, P.7)
- Academic Year
- Term/Semester
- Subject/Course Information
- Assessment Scores (Tests, Exams, Continuous Assessment)
- Grade/Results
- Attendance Records (Daily attendance, Absences, Reasons for absence)
- Promotion/Retention Status
- Academic Performance History

### 3. **Staff Information**
- Staff ID (unique identifier)
- Full Name
- Date of Birth
- Gender
- Nationality
- Contact Information (Phone, Email, Address)
- Employment Details:
  - Position/Role (Head Teacher, Deputy Head, Teacher, Support Staff)
  - Subject Specialization (for teachers)
  - Employment Date
  - Employment Status (Full-time, Part-time, Contract)
- Qualifications (Education Level, Certifications)
- Salary Information (if applicable for system design)
- Class/Subject Assignments

### 4. **Class and Subject Management**
- Class Information (Class Name, Level, Section if applicable)
- Class Capacity
- Subject/Course Details:
  - Subject Name
  - Subject Code
  - Class Level(s) where taught
  - Teacher Assignment
- Timetable Information (if available)

### 5. **Financial Records** (if applicable)
- Fee Structure (Tuition, Development, PTA, etc.)
- Payment Records:
  - Student ID
  - Payment Type
  - Amount Paid
  - Payment Date
  - Receipt Number
- Outstanding Balances
- Payment Methods

### 6. **Infrastructure and Resources**
- Classroom Information (Classroom Number, Capacity, Facilities)
- Library Resources (Books, Materials)
- Laboratory Equipment (if applicable)
- Sports Facilities
- Other Facilities (Computer Lab, etc.)

### 7. **Administrative Data**
- School Information:
  - School Name
  - School Code/Registration Number
  - Location/Address
  - Contact Information
  - Establishment Date
- Academic Calendar:
  - Term Dates
  - Holiday Periods
  - Examination Periods
  - Special Events

### 8. **Parent/Guardian Information**
- Guardian ID (if applicable)
- Full Name
- Relationship to Student (Father, Mother, Guardian, etc.)
- Contact Information (Phone, Email, Address)
- Occupation
- Emergency Contact Information

### 9. **Disciplinary Records** (if maintained)
- Incident Date
- Incident Type
- Description
- Action Taken
- Follow-up Status

### 10. **Health Records** (if maintained)
- Medical History
- Vaccination Records
- Allergies
- Emergency Medical Contacts
- Health Check-up Dates

---

## Data Collection Notes

### Priority Data (Essential for System Functionality):
1. Student basic information and enrollment data
2. Staff information and assignments
3. Class and subject structure
4. Academic records (current and historical)
5. Attendance records

### Secondary Data (Enhancement Features):
1. Financial records
2. Health records
3. Disciplinary records
4. Infrastructure details

### Data Format Preferences:
- **Dates:** Please provide in a consistent format (e.g., DD/MM/YYYY or YYYY-MM-DD)
- **Names:** Full names preferred (First, Middle, Last)
- **IDs:** If unique identifiers exist, please include them
- **Files:** If data is in Excel/CSV format, that would be helpful

### Data Privacy Considerations:
- We understand the sensitivity of student and staff data
- Data will be used solely for this academic project
- Appropriate security measures will be implemented
- Personal identifiers will be handled with confidentiality

## Questions for School Administration

To better understand the system requirements, please provide information on:

1. **Current Record-Keeping System:**
   - How are student records currently maintained? (Paper-based, Excel, other system?)
   - What are the main challenges with the current system?

2. **Key Processes:**
   - How is student enrollment handled?
   - How are examination results recorded and stored?
   - How is attendance tracked?
   - How are fees collected and recorded?

3. **Reporting Needs:**
   - What reports does the school administration need regularly?
   - What information do parents/guardians frequently request?

4. **User Roles:**
   - Who will use the system? (Administrators, Teachers, Support Staff)
   - What different access levels are needed?

5. **System Priorities:**
   - What are the most critical problems the system should solve?
   - What features would be most beneficial?
